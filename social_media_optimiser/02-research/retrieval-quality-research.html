Recommended Retrieval Architecture

1. Query Expansion

Rewrite the user topic into concise retrieval queries, exact-keyword probes, entity probes, and freshness probes. Keep the rewritten queries in the run event log.

2. Parallel Candidate Generation

Run BM25/full-text, dense pgvector, web search, metadata filters, and graph traversal in parallel. Do not trust a single retriever for source-backed answers.

3. Rank Fusion

Fuse sparse, dense, web, and graph candidates with Reciprocal Rank Fusion before expensive reranking. RRF avoids brittle score normalization across retrievers.

4. Rerank Top Candidates

Use cross-encoder reranking or late-interaction retrieval over the fused top-K. This is where we cut false positives before content generation.

5. Build Evidence Graph

Extract entities, relationships, claims, sources, dates, and artifact dependencies. Traverse local neighborhoods for exact support and global communities for coverage.

6. Coverage Gate

Every major claim must map to a source record, contradiction check, and freshness status. Unsupported claims stay marked unsupported and are blocked from final publishing.

Design Decisions

Postgres + pgvector remains the local memory base

Use pgvector for semantic memory and candidate retrieval because it keeps vectors with durable Postgres run/source/artifact state. Add HNSW indexes and metadata filters as corpus size grows.

RRF before reranking

Fuse retriever outputs before cross-encoder scoring. This makes the reranker see better candidates and reduces dependence on a single retrieval signal.

GraphRAG for multi-hop and corpus-level questions

Knowledge graphs are not a replacement for vector search. They add entity/relationship traversal and global summaries when semantic search cannot connect evidence across sources.

Evaluation is part of the harness

Track retrieval misses, weak sources, unsupported claims, stale evidence, and source diversity as run artifacts so agents improve over time.

Sources Used

  • pgvector official repository - Postgres vector similarity search, approximate/exact search, HNSW, vector types, and storage options.
  • Elasticsearch Reciprocal Rank Fusion docs - RRF combines top results from multiple retrievers without shared score semantics.
  • Microsoft GraphRAG docs - Knowledge graph extraction, community hierarchy, and graph-augmented query processes.
  • Microsoft Research GraphRAG project - GraphRAG, DRIFT search, LazyGraphRAG, and provenance-related research direction.
  • ColBERT paper - Late interaction preserves fine-grained query-document matching with faster retrieval than full cross-encoder scoring.
  • Anthropic Contextual Retrieval - Contextual embeddings, contextual BM25, hybrid retrieval, and reranking reduce failed retrievals.
  • Sentence Transformers CrossEncoder docs - Pairwise reranking is more accurate for relevance scoring but cannot precompute document embeddings.
  • OpenAI Retrieval docs - Query rewriting, vector stores, attribute filtering, and hybrid search weight controls.
