# Agent Studio System Design Vault

This is the separate Obsidian vault for system design canon, source ingestion, HLD/LLD, and architecture learning.

It is not the product app. It is not the existing `social_media_optimiser` planning vault. It is a research/design vault for long-running system design sources and their implications for Agent Studio.

## Operating Rules

- Use official, public, open-license, publisher-provided, author-provided, standards, paper, or user-confirmed lawful local sources.
- Do not use Anna's Archive, LibGen, Z-Library, or similar shadow-library downloads as acquisition sources.
- Do not auto-ingest PDFs whose filename indicates Anna's Archive, Z-Library, LibGen, or other likely unauthorized origins.
- For copyrighted books, store notes and design implications, not raw extracted book text.
- Keep raw extracts and lawfully provided PDFs outside tracked notes or in ignored local folders.
- Use this vault for system design notes; use generated HTML/JSON only as a companion inspection surface.

## Folder Map

- `00-index/` - maps of content and ingestion status.
- `01-sources/` - source inventory, legal status, and official/open source notes.
- `02-books/` - book-level reading maps and chapter-note stubs.
- `03-patterns/` - synthesized reusable architecture patterns.
- `04-agent-studio-implications/` - HLD, LLD, and product-specific decisions.
- `05-ingestion-runs/` - ingestion run logs and TODOs.
- `templates/` - reusable Obsidian note templates.
- `tools/` - local ingestion helpers.
- `output/viewers/` - generated JSON/HTML inspection companions.
