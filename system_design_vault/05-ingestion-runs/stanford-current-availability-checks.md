## Agent Studio Implications

- The datastore needs a separate availability layer before source ingestion. A route should not infer video coverage from a course listing.
- `source_available`, `future_pending`, `gated_private`, `public_listing_only`, `deep_read_note_created`, and `video_not_claimed` should be distinct status values.
- Source notes should record whether the evidence came from public lecture notes, a public video, a schedule listing, a primary paper, official docs, or a gated artifact.
- Promotion gates should block canon promotion for future or gated material unless there is already enough directly read official/open material from other eligible sources.
- Public-video coverage needs a separate `video_ingested` flag from `public_recording_visible`; a YouTube link alone is not evidence that the lecture has been understood.

## Datastore Objects

| Object | Job | Minimum fields |
|---|---|---|
| `source_availability_check` | Auditable check before reading or note creation | `check_id`, `source_ref`, `checked_at`, `checked_by`, `availability_status`, `public_materials_found`, `evidence_urls`, `next_check_after`, `notes` |
| `gated_video_record` | Tracks official recordings that exist but are not publicly usable | `video_record_id`, `course_ref`, `lecture_ref`, `gate_type`, `access_requirement`, `allowed_use_status`, `replacement_public_source_ids`, `status` |
| `future_source_watch` | Prevents premature notes for future-dated or placeholder material | `watch_id`, `source_ref`, `expected_date`, `current_public_status`, `last_checked_at`, `next_check_after`, `promotion_blocker` |
| `public_recording_visibility_check` | Separates schedule/recording-page visibility from actual watched-video ingestion | `visibility_check_id`, `course_ref`, `lecture_ref`, `recordings_page_url`, `visible_as_public_recording`, `video_watch_ingested`, `direct_full_watch_required`, `checked_at` |
