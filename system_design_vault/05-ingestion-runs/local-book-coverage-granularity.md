# Local Book Coverage Granularity

## Purpose

This audit separates local books with chapter-level canon coverage from local books with targeted book-slice synthesis. Both can be valid Agent Studio evidence, but they are not the same coverage claim. This note stores no raw book text or long excerpts.

## Current Counts

- Canon-ready local book records: 24.
- Chapter-sweep parent books: 8.
- Targeted parent books with chapter-level deepening: 13.
- Canon-ready chapter notes: 114.
- Full short-document canon notes: 1.
- Targeted book-slice canon notes without chapter-level deepening: 3.
- Deferred/private/excluded/duplicate local records: 10.
- Deferred local records that can become support material only through explicit trigger conditions are tracked in `05-ingestion-runs/deferred-local-corpus-queue.json`.

## Coverage Classes

| Class | Meaning | Use as evidence |
|---|---|---|
| `chapter_sweep` | The book has a chapter manifest and per-chapter compact notes. | Strongest local-book evidence for broad source coverage. |
| `targeted_book_slice_with_chapter_notes` | The book remains a targeted slice overall, but route-critical chapters have compact chapter notes. | Strong for the named chapters and valid for the broader targeted slice, but not a full-book sweep. |
| `full_short_document_pass` | The full local document is short enough to read end-to-end and has one compact synthesis note. | Stronger than a targeted slice for that file, but not a chapter-sweep book. |
| `targeted_book_slice` | The note directly reads and synthesizes selected sections relevant to Agent Studio. | Valid for the named route/design slice, but not a claim that every chapter was covered. |
| `question_backlog_ready` | Local material was converted into eval/concept backlog only. | Use to find review gaps, not as factual source canon. |
| `defer` / `defer_private_notes` / `exclude` / `avoid_double_ingestion` | Material is background, private, out of scope, or duplicate. | Do not use as architecture evidence unless status changes. |

## Agent Studio Implications

- A source-backed route may cite `targeted_book_slice` notes only for the route surface they explicitly cover.
- Chapter-sweep books can support broader book-level reasoning, but final design claims still need the relevant chapter note or cross-source canon note.
- Future ingestion should deepen a targeted slice into chapter coverage only when the book becomes central to a production route, eval suite, or datastore object family.
- The source-map should expose coverage granularity so downstream agents do not mistake slice notes for full-book coverage.
