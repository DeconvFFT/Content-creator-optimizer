# Deep Reading Coverage Ledger

This ledger tracks only direct-reading coverage. TOC-based or source-outline notes are not accepted as progress and should not be created.

## Status Values

- `pending_direct_read`: source is in scope but no direct-reading notes exist yet.
- `deep_read_pass_1`: source section was read directly and the note was revised from the source.
- `cross_check_pass_1`: direct-reading note has at least one official/open cross-check note but is not fully promoted to canon.
- `canon_ready`: note has been cross-checked against adjacent official docs/books and can support architecture decisions.
- `blocked`: cannot proceed without access, provenance, tooling, or design clarification.
